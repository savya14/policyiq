# PolicyIQ: Comprehensive Project Report

## 1. Project Overview
**PolicyIQ** is an advanced AI-powered RAG (Retrieval-Augmented Generation) assistant developed for Indian Oil Corporation Limited (IOCL) employees. Its primary objective is to streamline the compliance and regulatory checking process. It instantly answers complex questions concerning safety regulations, delegation of powers, HR policies, and internal procedures by retrieving context from a vast, vector-indexed corpus of official regulatory documents.

## 2. Technology Stack & Tools Used
The project utilizes a modern, robust tech stack split between a fast Python-based API and a responsive React frontend.

### 2.1 Backend (AI & Data Pipeline)
*   **FastAPI & Uvicorn**: Serves as the high-performance backend web framework to handle API requests and streaming responses.
*   **LangChain & LangSmith**: Orchestrates the RAG pipeline, manages prompt templates, orchestrates retrieval chains, and handles conversational memory.
*   **Groq API**: Provides lightning-fast LLM inference to generate answers based on retrieved context.
*   **FAISS (Facebook AI Similarity Search)**: Acts as the vector database (`index.faiss`) for highly efficient similarity search of document chunks.
*   **HuggingFace / Sentence Transformers**: Used to generate dense vector embeddings of the document chunks.
*   **PyPDFium2 & pdfplumber**: Employed for robust parsing and text extraction from raw PDF documents.

### 2.2 Frontend (User Interface)
*   **React (v18) & Vite**: Powers the Single Page Application (SPA), providing a fast development experience and optimized production build.
*   **Tailwind CSS**: Used for rapid, utility-first styling and a modern UI layout.
*   **Framer Motion**: Delivers smooth, micro-animations to enhance user experience and UI transitions.
*   **React-PDF**: Renders source document pages directly within the browser, allowing users to verify the AI's claims.
*   **React-Markdown**: Safely renders the Markdown-formatted responses generated by the LLM.

## 3. System Architecture & Capabilities
The system is designed around a highly accurate RAG architecture with a focus on enterprise reliability.

### 3.1 Document Pipeline & Deduplication
The corpus currently consists of **29 official regulatory documents** (including OISD, PNGRB, and PESO standards) parsed into **7,038 discrete chunks**. 
*   **Dual-layer Deduplication**: The system prevents redundant data by utilizing SHA-256 hashing for bitwise exact matches and MD5 registries to track canonical vs. non-canonical/superseded versions.
*   **Smart Chunking**: Documents are chunked using optimized parameters (`chunk_size=600`, `overlap=200`) which were empirically proven to yield up to 100% `Recall@10` in complex retrieval benchmarks.

### 3.2 Advanced Query Handling
The system handles not just "happy path" queries, but is strictly evaluated against complex edge cases:
*   **Cross-Document Queries**: Capable of synthesizing answers that require rules from both PESO and OISD.
*   **Substance Conflation Avoidance**: Trained to not confuse closely related terms (e.g., CNG vs. LPG storage rules).
*   **Conversational Memory**: Users can ask follow-up questions (e.g., "And what capacity?"), and the system retains context from the previous interaction.
*   **Absence Acknowledgment**: If a specific rule doesn't exist in the corpus, the system acknowledges this rather than hallucinating, offering the closest relevant standard instead.

## 4. How PolicyIQ Helps the User (Value Proposition)
PolicyIQ completely transforms how IOCL employees interact with compliance documents:

1.  **Instant Regulatory Answers**: Instead of manually reading through hundreds of pages of dense PDFs (e.g., the 267-page OISD-STD-144), an employee can simply ask "What is the minimum safe distance for a 20 Cu.Mt. LPG vessel?" and get the correct answer (15m) instantly.
2.  **Verifiable Citations**: Trust is critical in compliance. Every answer provided by PolicyIQ includes the exact source document and page number. The frontend UI allows the user to click the citation and view the original PDF page to verify the rule themselves.
3.  **Safety & Risk Mitigation**: By making safety rules highly accessible, employees are less likely to make dangerous assumptions. Emergency response protocols (like PNGRB ERDMP) and safety standards are just a prompt away.
4.  **Time Efficiency**: Tasks that previously required an expert to spend hours referencing multiple physical or digital manuals can now be completed in seconds, drastically improving operational efficiency.
5.  **Feedback-Driven Improvement**: Built-in feedback mechanisms (👍 / 👎) allow administrators to monitor queries and continually tune the retrieval parameters to capture the exact needs of the organization.
